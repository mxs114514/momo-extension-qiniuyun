# AI 同声传译助手技术实施指南

> 本文只记录长期稳定的架构原则、模块边界和技术约束。具体开发步骤、文件清单、
> 命令与验收流程见对应实施计划。

## 1. 实施原则

1. 实时翻译链路优先于历史记录和角色功能。
2. 音频来源、云服务协议、字幕状态和 UI 相互隔离。
3. 流式结果按句子覆盖，禁止把中间结果重复追加。
4. 所有音频、网络和订阅资源必须支持幂等清理。
5. 自动化测试不依赖真实麦克风和真实腾讯云服务。
6. 当前项目定位为个人比赛演示，凭证通过本地环境配置管理。

## 2. 总体架构

```text
UI
└── SpeechTranslationController
    ├── AudioSource
    ├── SpeechTranslateClient
    └── TranscriptStore
```

依赖方向固定为：

```text
UI → Controller → AudioSource / Client / Store
```

底层模块不能依赖 React，也不能直接修改 UI 状态。

## 3. 模块边界

### `AudioSource`

负责采集音频并输出统一 PCM 数据，不负责云服务连接。

```typescript
interface AudioSource {
  start(onChunk: (chunk: Int8Array) => void): Promise<void>
  pause(): Promise<void>
  resume(): Promise<void>
  stop(): Promise<void>
}
```

实现约束：

- 一期实现 `MicrophoneAudioSource`。
- 浏览器插件阶段增加 `TabAudioSource`。
- 不同音频源必须输出相同格式的数据。
- `pause()` 保留设备和处理节点，`stop()` 才彻底释放资源。

### `SpeechTranslateClient`

负责腾讯云实时语音翻译协议，不负责音频采集和 UI。

```typescript
interface SpeechTranslateClient {
  connect(): Promise<void>
  sendAudio(chunk: Int8Array): void
  stop(): void
  close(): void
}
```

主要职责：

- 构造签名和 WebSocket 地址。
- 发送 PCM 数据。
- 解析翻译结果和服务错误。
- 正常停止时等待会话最终响应。

### `TranscriptStore`

负责维护双语字幕数据：

```typescript
interface TranslationSentence {
  id: string
  sourceText: string
  targetText: string
  startTime: number
  endTime: number
  isFinal: boolean
}
```

处理规则：

- 使用 `sentence_id` 作为句子标识。
- 同一句的新响应覆盖旧响应。
- 保留英语原文和中文译文。
- 一期 UI 只展示中文译文。
- Store 对外返回副本，调用方不能修改内部状态。

### `SpeechTranslationController`

负责协调音频源、翻译客户端和字幕 Store：

- 管理状态机。
- 控制何时发送音频。
- 处理开始、暂停、继续和停止。
- 统一转换错误信息。
- 统一清理资源。
- 向 UI 提供可订阅的状态快照。

## 4. 音频约束

腾讯云输入统一为：

| 项目 | 要求 |
| --- | --- |
| 采样率 | 16000Hz |
| 采样精度 | 16bit |
| 字节序 | 小端 |
| 声道 | 单声道 |
| 格式 | PCM |
| 建议分包 | 约 200ms，6400 字节 |

音频处理使用 Web Audio API 和 AudioWorklet。采集模块负责降采样、PCM 转换和
固定大小分包。

## 5. 腾讯云协议约束

使用腾讯云实时语音翻译 WebSocket，语言方向为英语到中文。

客户端需要同时处理：

- `source_text`：英语原文。
- `target_text`：中文译文。
- `sentence_id`：句子标识。
- `sentence_end`：当前句完成。
- `final`：整个会话结束。

`source_text` 和 `target_text` 是当前句的完整内容，不是增量片段。

停止时先阻止新音频进入，再发送结束消息，等待会话最终响应；超时后允许强制
关闭连接。

仓库中的普通 ASR SDK只作为录音和协议实现参考，不作为实时语音翻译业务依赖。

## 6. 状态机

```text
idle
  → requesting-permission
  → connecting
  → translating
  → paused
  → translating
  → stopping
  → idle

任意状态 → error
```

规则：

- 只有 `translating` 可以暂停。
- 只有 `paused` 可以继续。
- 暂停期间停止发送音频，但保留当前 WebSocket 会话。
- 停止操作需要释放音频资源并正常结束网络会话。
- 网络、权限或协议错误进入 `error`。
- 所有重复操作必须保持幂等。

暂停时间过长可能导致腾讯云关闭连接，此时提示用户重新开始，不自动创建新会话。

## 7. UI 边界

一期 UI 负责：

- 开始、暂停、继续和停止操作。
- 展示当前状态。
- 展示中文字幕。
- 展示错误和安全提示。

UI 不负责：

- PCM 转换。
- WebSocket 消息解析。
- 字幕合并。
- 直接读取或保存云服务密钥。

布局和常规交互样式优先使用 Tailwind CSS。全局 CSS 只保留主题变量、基础元素
样式和工具类不适合表达的少量规则。

## 8. 错误处理

错误至少分为：

| 类型 | 示例 |
| --- | --- |
| 配置错误 | 缺少 AppID 或密钥 |
| 权限错误 | 用户拒绝麦克风权限 |
| 设备错误 | 无可用麦克风 |
| 能力错误 | 浏览器不支持所需音频 API |
| 鉴权错误 | 腾讯云拒绝签名 |
| 服务错误 | 腾讯云返回非零错误码 |
| 网络错误 | WebSocket 异常关闭 |
| 超时错误 | 停止后未收到最终响应 |

底层错误需要转换为可理解的中文提示，但日志和 UI 不能包含 SecretKey 或完整
签名 URL。

## 9. 测试边界

### 单元测试

覆盖：

- PCM 转换和分包。
- 签名参数排序与编码。
- WebSocket 协议解析。
- 字幕覆盖和排序。
- 状态机和资源清理。

### 组件测试

覆盖：

- 状态与按钮变化。
- 只展示中文译文。
- 中间字幕覆盖。
- 错误提示和重试。

### 端到端测试

覆盖页面主要交互，不连接真实腾讯云或真实麦克风。

真实麦克风权限、腾讯云账号、网络延迟和翻译效果通过手工联调验证。

## 10. 凭证边界

当前项目用于个人比赛演示，前端直连腾讯云服务，凭证通过本地环境变量配置。

- 真实密钥不得提交到 Git。
- 密钥不得写入源码。
- SecretKey 和完整签名 URL 不得输出到日志。

## 11. 阶段迁移

### 浏览器插件阶段

保留：

- `AudioSource` 接口。
- 翻译客户端。
- 字幕 Store。
- Controller 状态机。

替换或增加：

- `MicrophoneAudioSource` 替换为 `TabAudioSource`。
- 普通页面拆分为 Popup、Side Panel 和 Content Script。
- 使用 Offscreen Document 承载持续音频任务。
- 使用 Chrome Messaging 进行跨上下文通信。

### 二期以后

- 历史记录使用 IndexedDB 和 Dexie。
- 导出使用 Blob API。
- AI 翻译官复用现有字幕 Store。
- LLM 和 Live2D 必须作为可降级模块，不能阻塞实时翻译链路。
