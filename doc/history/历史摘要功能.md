# DeepSeek 历史记录摘要改造计划

## Summary

将历史记录摘要从“第一条有效中文字幕前 80 字”升级为 DeepSeek 生成摘要。摘要在用户选择“保存并停止”后、后台保存历史记录时生成；DeepSeek 不可用或未配置时自动回退到当前本地摘要逻辑，保证历史记录保存不被 AI 服务阻塞失败。

参考 DeepSeek 官方文档：DeepSeek API 使用 OpenAI 兼容格式，当前 Chat API endpoint 为 `https://api.deepseek.com/chat/completions`，当前模型为 `deepseek-v4-flash` / `deepseek-v4-pro`，旧 `deepseek-chat` / `deepseek-reasoner` 将于 2026-07-24 废弃；Chat Completion 支持 `messages`、`model`、`thinking`、`max_tokens`、`temperature` 等参数。来源：[DeepSeek First API Call](https://api-docs.deepseek.com/)、[Create Chat Completion](https://api-docs.deepseek.com/api/create-chat-completion)。

## Key Changes

- 新增 DeepSeek 配置：
  - `.env.example` 增加 `VITE_DEEPSEEK_API_KEY=`、`VITE_DEEPSEEK_MODEL=deepseek-v4-flash`。
  - 新增解析函数，未配置 API Key 时返回“禁用 DeepSeek 摘要”，不抛错。
  - `public/manifest.json` 的 `host_permissions` 增加 `https://api.deepseek.com/*`。

- 新增摘要生成边界：
  - 在 `src/features/session-history` 下增加 `summary-generator` 相关模块。
  - 定义接口：`generateSummary(sentences: TranslationSentence[]): Promise<string>`。
  - 默认本地实现保留当前逻辑：第一条有效中文字幕 `trim().slice(0, 80)`。
  - DeepSeek 实现通过 `fetch` 调用 Chat Completions，不引入 OpenAI SDK。

- 改造历史记录保存：
  - `SessionHistoryStore` 构造函数支持注入 `summaryGenerator`。
  - `saveSession()` 过滤有效句子后，优先调用注入的摘要生成器。
  - DeepSeek 请求失败、超时、返回空内容、响应结构异常时，回退本地摘要。
  - `default-history-store.ts` 根据 env 创建 DeepSeek 摘要生成器；未配置 key 时使用本地摘要。

## DeepSeek Request Shape

- 使用 `POST https://api.deepseek.com/chat/completions`。
- Header：
  - `Authorization: Bearer ${VITE_DEEPSEEK_API_KEY}`
  - `Content-Type: application/json`
- Body 默认：
  - `model: import.meta.env.VITE_DEEPSEEK_MODEL || 'deepseek-v4-flash'`
  - `thinking: { type: 'disabled' }`
  - `temperature: 0.2`
  - `max_tokens: 120`
  - `stream: false`
  - `messages`：
    - system：要求用中文生成 1 句学习记录摘要，不超过 60 个汉字，不输出解释。
    - user：传入最多前 20 条有效中文译文，每条一行。
- 解析 `choices[0].message.content.trim()` 作为摘要。

  - generator 抛错或返回空字符串时，回退第一句前 80 字。

## Assumptions

- 当前项目仍是个人比赛演示，DeepSeek API Key 可通过本地 `.env.local` 配置；真实 key 不提交 Git。
- 第一版只在保存历史记录时生成摘要，不在列表页异步补生成，也不增加“重新生成摘要”按钮。
- 摘要生成失败不提示用户，静默回退本地摘要，避免影响停止翻译和保存记录流程。
- 默认模型使用 `deepseek-v4-flash`，不用即将废弃的 `deepseek-chat`。
